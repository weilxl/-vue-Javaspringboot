# 后端八股文（结合本项目）

## Spring Boot 原理
Spring Boot 通过起步依赖和自动配置减少 XML 与手写配置，内嵌 Tomcat，运行一个 main 方法即可启动 Web 应用。本项目 EfficiencyApplication 上标 @SpringBootApplication，启动后自动扫描本包及子包下的 Bean、配置 MVC、连接数据源等，就像一键启动的「集成好的服务器」。

## MVC 架构
请求先到 DispatcherServlet，再根据 URL 映射到对应 Controller 方法，方法里调 Service 做业务、用 Entity/DTO 承载数据，最后返回 JSON。本项目 AuthController、TodoTaskController 等即 Controller 层，只做参数接收与结果封装，业务在 Service 中，符合单一职责。

## MyBatis 原理
MyBatis 负责对象与 SQL 的映射，通过 Mapper 接口与 XML（或注解）定义 SQL，执行后把 ResultSet 封装成实体。本项目用 MyBatis Plus，在 Mapper 上继承 BaseMapper 即拥有单表 CRUD，复杂统计在 Service 里用 stream 或额外 SQL 完成，如 StatsController 的 dashboard 聚合。

## JWT 原理
JWT 是「头部+载荷+签名」的字符串，载荷里存 userId、username、roleCode 等，服务端用密钥签名，客户端携带 token 请求时服务端验签并解析出用户信息，无需在服务端存 session。本项目 JwtTokenUtil 负责生成与解析，拦截器在受保护路径上校验 token 并把用户信息放入 UserContext（ThreadLocal），供 Controller 使用。

## RESTful 设计
用 HTTP 方法表达语义：GET 查询、POST 新增、PUT 更新、DELETE 删除；URL 表示资源，如 /todo-task/list、/todo-task/add、/schedule/delete/{id}。本项目接口按此设计，前后端约定一致，便于理解和扩展。

## 数据库索引
对常作为查询条件的列建索引可加快查询。本项目 id 为主键自带聚簇索引；若按 username、due_date、start_time 等查得多，可在对应列上建索引。教学项目数据量小，未显式建索引也可，面试时可说「上线前会对高频查询字段加索引」。

## 事务
多步写操作需要要么全成功要么全回滚时，用 @Transactional。本项目单表单步操作多，未显式用事务；若以后做「删待办同时写日志」等多表操作，可在 Service 方法上加 @Transactional 保证一致性。
