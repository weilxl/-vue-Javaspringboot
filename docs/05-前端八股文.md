# 前端八股文（结合本项目）

## Vue2 原理
Vue2 通过 Object.defineProperty 对 data 做响应式劫持，配合 Watcher 和 Dep 做依赖收集与派发更新，从而在数据变化时自动更新视图。本项目里表格、表单、图表都依赖这一套：改 list 或 form，界面自动刷新，就像改 Excel 单元格后图表自动更新。

## 生命周期
created 时数据已可访问但 DOM 未挂载；mounted 时 DOM 已挂载，适合做 ECharts 等依赖容器的初始化。本项目 DashboardView 在 mounted 里调 loadStats，在 $nextTick 里渲染两个图表，确保容器已存在且数据已返回。

## 路由原理
Vue Router 通过 mode 为 history 或 hash 改变 URL 与组件的对应关系，不刷新整页即可切换视图。本项目用 history 模式，路由守卫在 beforeEach 里检查 token，无 token 访问业务页会重定向到登录页，有 token 访问登录页会重定向到首页，保证「未登录进不来、已登录不重复进登录页」。

## Vuex
Vuex 集中管理 token 和 userInfo，通过 mutations 修改状态并同步到 localStorage，这样多页面、多组件共享同一份登录态，避免每个页面各自读本地存储导致不一致。就像公司把员工信息放在统一档案室，各部门都去那里查。

## Axios
Axios 负责发 HTTP 请求。本项目里用实例配置 baseURL 和拦截器：请求拦截器统一加 Authorization，响应拦截器判断 code 与 401，把业务错误和未登录统一处理，页面只关心「请求成功」的分支，减少重复代码。

## Token 鉴权
登录后把服务端返回的 token 存到 Vuex 和 localStorage，之后每次请求在 Header 里带 Authorization（Bearer + token）。后端用拦截器解析 token，合法则放行并注入用户信息，不合法或过期则返回 401，前端拦截器收到 401 后清空登录态并跳转登录页。就像进门领卡、出门还卡，中间每次进房间都要刷卡。

## ECharts
ECharts 是图表库，需要先对 DOM 容器执行 echarts.init，再 setOption 传入配置。本项目在首页用两个容器分别渲染「待办状态分布」柱状图和「优先级分布」饼图，数据来自 /stats/dashboard 的聚合结果，在 mounted 且数据加载完成后的 $nextTick 里初始化，避免容器未渲染或数据未到位。
