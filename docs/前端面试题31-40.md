# 前端面试题 31-40

## 31. 说说 slot 的用法
slot 是插槽，让父组件把内容「塞进」子组件指定位置。默认 slot、具名 slot（如 slot="header"）、作用域 slot（子传数据给父，如 slot-scope="scope"）。本项目 LayoutView 里 el-main 用默认 slot 放 router-view；很多 Element 组件用 slot="header"、slot="footer" 自定义头尾，表格里用 slot-scope 拿到当前行数据做操作列。

## 32. ref 和 $refs 怎么用？
在标签或组件上写 ref="xxx"，通过 this.$refs.xxx 拿到 DOM 或组件实例。常用于：拿表单调 validate、拿 DOM 做 ECharts 容器、调子组件方法。本项目登录页 ref="loginForm"，提交时 this.$refs.loginForm.validate() 触发表单校验。就像给柜子贴标签：之后要说「打开贴着『表单』的那个」就能找到。

## 33. 为什么 data 要写成函数？
组件可能被复用多次，若 data 是对象则多个实例共享同一引用，改一个会牵连其他。写成函数则每次创建组件实例都会执行一次，返回新的对象，保证每个实例一份独立 data。本项目每个页面的 data() 都 return 一个新对象，符合规范。

## 34. 如何理解 Vue 的「虚拟 DOM」？
用 JS 对象描述真实 DOM 结构，先对比新旧虚拟 DOM（diff），再只把变化的部分更新到真实 DOM，减少直接操作 DOM 的次数和范围。这样既能声明式写「界面长什么样」，又保证性能。就像先画草图再施工：草图改起来快，改完再按草图改真实建筑，避免乱拆。

## 35. 什么是「同步」和「异步」更新 DOM？
Vue 会把同一轮里的多次数据变更合并成一次 DOM 更新（异步），所以改完 data 立刻读 DOM 可能是旧的。用 $nextTick 可以等到这次更新完成后再执行。本项目在请求返回后改 stats，紧接着 $nextTick 里画图，保证 DOM 已按新 stats 渲染。

## 36. 如何做「全局」的请求错误提示？
在 Axios 响应拦截器里，对非 2xx 或 res.code !== 200 的情况，统一弹 Message 或 Toast，再 return Promise.reject，这样业务里可以只写 .then，.catch 只做补充处理或不再写。本项目在拦截器里对 401 做跳转，对其它错误可在拦截器里 message.error(res.message)，业务代码更简洁。

## 37. Vue 中如何做「按需加载」组件？
用 () => import('@/views/xxx.vue') 做路由组件懒加载，或对非首屏的大组件用 import() 动态加载。本项目为减少 chunk 加载问题改为直接 import，若环境稳定可以恢复懒加载以减小首包。就像图书馆：按需加载是「用哪本取哪本」，直接导入是「常用书都放桌上」。

## 38. 如何理解「声明式」和「命令式」？
命令式是「一步步告诉电脑怎么做」（如先 getElementById 再 innerHTML=）；声明式是「描述最终长什么样」（如 template 里写 {{ name }}），框架负责把状态变成界面。Vue 的 template + data 就是声明式，我们只关心数据，不手写 DOM 操作。就像点菜：声明式是「我要宫保鸡丁」，命令式是「先倒油再下料再…」。

## 39. 移动端和 PC 端共用接口要注意什么？
接口路径、参数、返回格式一致；注意日期时间格式（如统一 yyyy-MM-dd HH:mm:ss）；注意权限（同一 token 两端都能用）；大列表可做分页或懒加载。本项目移动端与后台共用 /todo-task、/schedule 等接口，日期用 value-format 统一，无额外适配。

## 40. 如何理解「组件化」？
把页面拆成可复用的块（组件），每个组件有自己的 template、data、逻辑，通过 props 和事件与外界通信。这样结构清晰、可复用、易测。本项目 LoginView、LayoutView、TodoTaskView、TextTabBar 等都是组件，组合成完整应用。就像乐高：一块块拼起来，每块可单独设计、反复使用。
