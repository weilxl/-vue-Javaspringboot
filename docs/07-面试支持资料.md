# 面试支持资料

## 简历项目描述（可直接粘贴）
个人效率管理系统：前后端分离的全栈项目，包含后台管理端（Vue2 + Element UI）与移动端（Vue2 Web）。后端使用 Spring Boot + MyBatis Plus + JWT + MySQL，实现用户登录注册、待办任务与日程安排的 CRUD、角色权限控制及首页统计图表接口；前端实现路由守卫、Token 鉴权、ECharts 数据可视化，移动端与后台共用接口实现数据同步。项目具备完整接口文档（Swagger）与数据库设计，适合作为实习简历项目展示。

## 项目亮点
- 前后端分离 + JWT 无状态认证，接口规范、Swagger 文档完整。  
- 双端复用同一后端（后台管理 + 移动端），体现接口设计与多端协作。  
- 首页 ECharts 展示待办状态与优先级分布，体现数据统计与可视化。  
- 路由守卫、请求拦截器统一处理 401，权限与鉴权流程清晰。

## 面试官可能提问
- 登录流程 / JWT 怎么用的？  
- 为什么用 Vuex 存 token？  
- 路由守卫做了什么？  
- 如何保证前后端日期格式一致？  
- Swagger 和 Spring Boot 2.6 遇到过什么问题？  

## 推荐回答要点
- 登录：前端调 /auth/login，后端校验用户密码后生成 JWT 返回，前端存 Vuex 和 localStorage，后续请求 Header 带 Token，后端拦截器解析并注入用户信息。  
- Vuex：统一存 token 和 userInfo，避免多页面各自读本地导致不同步，且便于响应拦截器在 401 时统一清空。  
- 路由守卫：beforeEach 里根据 token 判断，无 token 访问业务页跳登录，有 token 访问登录页跳首页。  
- 日期：前后端约定 yyyy-MM-dd HH:mm:ss，前端日期选择器用 value-format 输出该格式。  
- Swagger：Spring Boot 2.6+ 默认 PathPattern 与 Swagger 3 不兼容，需配置 matching-strategy 为 ant_path_matcher。

## 项目介绍稿（1～2 分钟）
这是一个个人效率管理系统，主要做待办任务和日程安排两块业务。技术栈是 Spring Boot 做后端，Vue2 + Element UI 做后台管理，还有一个 Vue2 的移动端，和后台共用一套接口。我负责了从建表、接口设计到前后端联调的完整流程：比如用 JWT 做登录认证，用路由守卫和请求拦截器做权限和 401 处理，首页用 ECharts 展示待办状态和优先级的统计图。过程中遇到过 Swagger 和 Spring Boot 路径匹配的兼容问题，通过改配置为 ant_path_matcher 解决；前后端日期格式也统一成 yyyy-MM-dd HH:mm:ss，避免提交时报错。整体上是一个结构清晰、可直接运行的全栈小项目，适合用来展示前后端分离和基础权限与统计能力。
