# 后端面试题 11-20

## 11. Spring 的依赖注入有哪些方式？
构造器注入、setter 注入、字段注入（@Autowired 在字段上）。推荐构造器注入：依赖明确、便于测试、不可变。本项目 Controller、Service 都用构造器注入（final 字段 + 构造函数），没有用 @Autowired 字段注入，符合常见最佳实践。

## 12. 如何理解「分层」？Controller、Service、Mapper 各做什么？
Controller 只做接收请求、调 Service、返回结果，不写业务逻辑；Service 做业务逻辑、事务编排、调 Mapper 或其它 Service；Mapper 只做数据访问。这样职责清晰，改业务只动 Service，改表结构主要动 Mapper。本项目 TodoTaskController 调 TodoTaskService，Service 调 TodoTaskMapper，分层明确。

## 13. 密码为什么要加密存储？本项目用的什么？
明文存密码一旦数据库泄露，所有用户密码暴露。用不可逆摘要（如 MD5）或加盐哈希（如 BCrypt）存储，登录时对用户输入的密码做同样运算再比对。本项目用 MD5 存密码（教学用），生产环境更推荐 BCrypt 等带盐哈希，防止彩虹表破解。

## 14. Swagger 是什么？怎么集成？
Swagger 是 API 文档与调试工具，根据 Controller 上的注解生成接口说明，并提供页面直接调接口。集成：加 springfox-boot-starter 依赖，写一个配置类 @Configuration 建 Docket、指定扫描包、配置 Authorization。本项目集成后访问 /api/swagger-ui/index.html 即可，需注意 Spring Boot 2.6+ 要配 ant_path_matcher。

## 15. 什么是 RESTful？本项目哪些接口符合？
RESTful 用 HTTP 方法表达操作：GET 查、POST 增、PUT 改、DELETE 删，URL 表示资源。本项目 /todo-task/list GET、/todo-task/add POST、/todo-task/update PUT、/todo-task/delete/{id} DELETE 即符合。资源是 todo-task，操作靠方法区分，一目了然。

## 16. 为什么需要 CORS 配置？
浏览器同源策略下，前端（如 localhost:8080）请求后端（如 localhost:8082）会跨域，后端需返回允许该源的 CORS 头，浏览器才肯把响应交给前端。本项目 WebConfig 里 addCorsMappings 允许相应 origin、methods、headers，方便开发联调。生产环境应限制具体域名，不要用 *。

## 17. 事务 @Transactional 什么时候用？
当一次业务要执行多条写操作，要求「要么全成功要么全回滚」时用。在 Service 方法上加 @Transactional，异常时自动回滚。本项目当前以单表单步操作为主，未显式加事务；若以后做「删待办并写日志」等多步写，应在 Service 方法上加注解。

## 18. 如何理解「无状态」？
服务端不存用户会话，每次请求自带身份信息（如 JWT），服务端只验签、解析用户即可，不依赖内存或 Redis 里的 session。这样水平扩展时不用考虑 session 粘滞，适合分布式。本项目即无状态：登录后只发 token，后续请求都带 token，后端不存会话。

## 19. 日期时间格式为什么要统一？
前端传字符串，后端用 LocalDateTime 等接收，若格式不一致（如前端只传日期、后端要日期时间）会反序列化失败。本项目约定 yyyy-MM-dd HH:mm:ss，前端日期时间选择器用该 value-format，后端 Jackson 已配置该格式，避免 400 错误。就像约好「都按北京时间」才不会搞错开会时间。

## 20. 如何设计「角色权限」？
常见 RBAC：用户-角色-权限。用户表有 role_code，角色表有 permissions（可存字符串或关联权限表）。接口里根据当前用户角色判断能否访问：如 RoleController 里只有 ADMIN 能查角色列表。本项目是简化版：角色和权限在 sys_role 里，接口里用 UserContext.get().getRoleCode() 判断即可。
