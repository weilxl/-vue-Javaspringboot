# 开发执行记录（开发日志）

## 需求分析
目标：做一个「个人效率管理系统」用于实习简历、技术学习与面试准备。要求前后端分离，包含后台管理 Web 与移动端 App，登录注册、权限控制、MySQL、JSON 接口，风格参考若依、主题色海洋清风，页面简洁、少图、优先 UI 组件与 Element 图标，无外键、初始有模拟数据，admin/admin123 可登录。

## 技术选型
- 后端：Spring Boot 2.7、MyBatis Plus、JWT、Swagger、MySQL。  
- 后台：Vue2、Element UI 2.9.2、Vuex、Vue Router、Axios、ECharts，包管理 pnpm。  
- 移动端：Vue2 纯 Web（非 Uni-app），Element UI、文字底部导航，HBuilder 运行到浏览器。  

选型理由：技术栈常见、资料多，适合零基础在三天内吃透并写在简历上。

## 开发步骤
1. 建项目根目录与根 POM，保证 Maven 可识别；编写 request.txt 固定 JDK8、Element 2.9.2 等版本。  
2. 后端：实体（SysUser、SysRole、TodoTask、Schedule）、Mapper/Service/Controller、JWT 与 Swagger 配置、init.sql（含 3～5 条/表初始数据）。  
3. 后台前端：登录、布局、首页两图（待办状态、优先级分布）、待办管理页、日程管理页，路由直接导入、海洋清风主题。  
4. 移动端：登录、注册、待办列表+新增弹窗、日程列表+新增弹窗、我的；底部文字 Tab，与后台共接口。  
5. 文档：启动环境、数据库设计、开发日志、学习资料、八股文、面试支持及面试题 5×2 份。

## 问题与解决
- **Swagger 与 Spring Boot 2.6+**：配置 `spring.mvc.pathmatch.matching-strategy: ant_path_matcher`。  
- **JWT 密钥长度**：使用至少 32 字符的 secret，满足 HMAC-SHA256 要求。  
- **日期时间格式**：前后端约定 `yyyy-MM-dd HH:mm:ss`，日程表单项使用 datetime 选择器。  
- **ESLint**：不使用 @babel/eslint-parser，用标准 ESLint + plugin:vue，避免 pnpm 下 Babel 组合报错。  
- **路由**：改为直接 import 组件，避免 chunk 加载失败。

## 优化过程
- 统一页面命名为 *View.vue；图标仅用约定集合；App 底部纯文字导航。  
- 请求头统一携带 Bearer Token；数据库账号密码按约定 root/123456。

## 最终成果
可运行的最小完整项目：后端 8082、Swagger 可访问、admin/admin123 登录成功；后台三页（首页两图+待办+日程）；App 五页（登录、注册、待办、日程、我的），数据与后台同步，适合作为简历项目与面试讲解素材。
