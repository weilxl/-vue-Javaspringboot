# 前端面试题 0-10

## 1. Vue2 为什么适合做企业后台？
Vue2 生态成熟、Element UI 组件全、资料多，适合快速交付企业后台。本项目用 Vue2 + Element 快速实现登录、表格、弹窗、图表，就像先用成熟工具搭好骨架再做业务装修，风险小、节奏稳。生活中好比装修时先买标准尺寸的橱柜，再按需求微调，比从零打家具更省事。

## 2. Vue2 响应式的核心机制是什么？
通过 Object.defineProperty 对 data 的属性做 getter/setter 劫持，再配合 Dep 与 Watcher 做依赖收集和派发更新：你改数据时 setter 触发，通知依赖该数据的 Watcher 更新视图。可以类比成 Excel：改某个单元格，所有引用该单元格的公式和图表会自动重算、重画，Vue 的「单元格」就是 data 里的属性。

## 3. 为什么要用路由守卫？
路由守卫能在「进入页面前」统一做判断，比如是否已登录、是否有权限，避免每个页面重复写鉴权逻辑。本项目中 beforeEach 里检查 token：没有 token 访问业务页就重定向到登录页，有 token 还访问登录页就重定向到首页。就像大楼门禁：进每层前都要在一楼闸机统一验卡，而不是每层各设一个保安重复查。

## 4. 为什么把懒加载改成直接导入？
懒加载会把组件打成单独 chunk，依赖路径和部署环境，容易在部分环境下出现 chunk 加载 404。你的需求更偏向「稳定可运行、适合演示」，直接 import 组件则打包进主 bundle，不依赖异步加载，更适合教学和实习展示。好比出差：懒加载像「到了再买日用品」，直接导入像「行李箱里都带齐」，后者更稳。

## 5. Vuex 在本项目中解决了什么问题？
统一存 token 和 userInfo，所有页面从同一份 state 读，登录时 setAuth、退出时 clearAuth，并同步写 localStorage。这样不会出现「A 页面已退出、B 页面还拿着旧 token」的不一致。生活例子：公司把员工信息放在统一档案室，各部门都去那里查，而不是每个部门各存一份，避免信息不同步。

## 6. Axios 拦截器最佳实践是什么？
请求拦截器：统一给 header 加 Authorization（Token），这样业务代码不用每次手写。响应拦截器：统一判断 res.code 和 401，业务错误弹 message，401 则清空登录态并跳转登录页。这样页面只关心「请求成功」的分支，错误和鉴权逻辑集中收口，维护成本低。就像快递：所有包裹统一贴单、统一处理退件，收件人只关心「收到」这一种结果。

## 7. 如何理解前端权限控制？
前端权限本质是「体验上的控制」：根据角色隐藏菜单、禁用按钮，避免用户误点无权限功能。真正安全必须靠后端：后端每个接口都校验身份和权限，前端只是「不给你看」，不能防止直接调接口。两者配合才完整：前端像「提示牌告诉你此路不通」，后端是「真有一道门拦着」。

## 8. ECharts 为什么在 mounted 里初始化？
图表需要绑定真实 DOM 容器，并依赖容器的宽高来渲染。created 时 DOM 还未挂载，拿不到容器；mounted 时组件已挂载，能拿到 div#taskChart 等节点。本项目在 mounted 里先请求数据，在 $nextTick 里再 echarts.init 和 setOption，确保既有 DOM 又有数据。就像先摆好画框、再往里面装画。

## 9. 前后端分离下如何解决跨域？
开发时常用两种方式：一是后端配置 CORS，允许前端所在域名访问；二是前端 devServer 配置 proxy，把 /api 转发到后端地址，浏览器只和同源的前端服务器通信，由前端服务器转发到后端，从而没有跨域。本项目用 proxy，前端请求 /api/xxx 被转发到 localhost:8082/api/xxx，简单稳定。

## 10. 如何避免组件命名混乱？
制定并遵守统一规范。本项目约定：页面级组件一律用 *View.vue（如 LoginView、TodoTaskView），一眼能区分「整页」和「页面里的子组件」，便于协作和路由配置。生活中好比文件命名：报告都用「日期-主题.pdf」，就不会和随便起的「新建文档1」混在一起。
