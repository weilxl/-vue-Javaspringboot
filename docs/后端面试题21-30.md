# 后端面试题 21-30

## 21. 什么是「自动配置」？
Spring Boot 根据 classpath 里的依赖和配置文件，自动创建 Bean、配置数据源等，无需手写大量配置。例如加了 mybatis-plus-boot-starter 就自动配好 SqlSessionFactory、Mapper 扫描等。就像「智能家居」：检测到有空调就自动配好遥控，不用你一个个接线。

## 22. @RequestBody 和 @RequestParam 的区别？
@RequestBody 把请求体（通常 JSON）反序列化成对象，用于 POST/PUT 的 body；@RequestParam 取 URL 查询参数或表单字段，如 ?id=1。本项目登录用 @RequestBody LoginRequest，删除用 @PathVariable Long id，符合 REST 习惯。

## 23. 如何防止 SQL 注入？
用预编译：MyBatis 里用 #{} 占位符会生成 PreparedStatement，参数不会拼进 SQL；不要用 ${} 拼字符串。本项目 Mapper 用 Plus 提供的方法或 Lambda 条件，均为参数化查询，不会出现把用户输入直接拼进 SQL 的情况。

## 24. 索引是什么？什么时候建？
索引是帮助数据库快速定位行的数据结构，类似书的目录。对常作为查询条件、排序、分组的列建索引能明显加速；写多读少或数据量很小时可少建。本项目教学用、数据少，未显式建索引；面试可说「上线前会对 username、due_date 等高频查询字段加索引」。

## 25. 什么是「N+1 查询」？如何避免？
一次查主表得到 N 条，再对每条查一次关联表，共 1+N 次查询。避免：用 join 一次查出来、用 MyBatis 的 association/collection 做结果映射、或在 Service 里先批量查关联再组装。本项目当前无复杂关联，暂无 N+1 问题；若以后「待办列表带创建人信息」可一次 join 或批量查用户。

## 26. 如何理解「接口幂等」？
同一请求多次执行，效果和执行一次相同。如「扣款」不幂等，重复提交会多扣；「根据订单号扣款」可设计成幂等。实现方式：唯一键、状态机、token 等。本项目增删改若前端重复提交可能产生重复数据，严格场景可加唯一约束或前端防重复提交。

## 27. 日志级别怎么用？
通常用 info 记录关键流程、error 记录异常；开发时可把某包设为 debug 便于排查。本项目 application.yml 里 com.efficiency: info，生产可改为 warn 减少日志量。就像监控：平时看概要（info），出问题再调高清晰度（debug）。

## 28. 如何理解「前后端分离」下的职责？
后端只提供 JSON 接口，不负责渲染页面；前端负责页面、路由、调用接口并展示。这样后端可被多端复用（Web、App、小程序），前端可独立迭代。本项目后端只返回 ApiResponse 包装的 data，前端负责登录页、表格、图表等所有界面。

## 29. 为什么 JWT 要设过期时间？
无限期 token 一旦泄露可一直被冒用。设过期时间（如 24 小时）后，过期则需重新登录，降低风险。本项目 jwt.expire-hours: 24，解析时 isExpired 会判断，过期返回 401，前端跳转登录。

## 30. 如何做「分页」？
数据库用 LIMIT offset, size；MyBatis Plus 有 Page 和分页插件，Service 里 page(condition) 即可。前端传 pageNum、pageSize，后端返回当前页数据和 total。本项目当前列表全部返回未做分页，若数据量大可加 PageHelper 或 Plus 的分页，接口增加分页参数和返回结构即可。
