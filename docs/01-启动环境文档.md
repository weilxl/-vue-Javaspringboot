# 启动环境文档（个人效率管理系统）

## 1. 先装什么

本项目分三端：后端 Spring Boot、后台 Vue2 + Element UI、移动端 Vue2（纯 Web，非 Uni-app）。可以把它理解成「公司三件套」：后端是数据中心，后台是管理大屏，移动端是手机上的办事入口。建议按顺序安装：**JDK8**、**Maven**、**MySQL**、**Node.js 16+**、**pnpm**。装好后执行 `java -version`、`mvn -v`、`mysql --version`、`node -v`、`pnpm -v` 检查，版本以根目录 `request.txt` 为准，大体一致即可跑通。

## 2. 后端运行方式

进入 **backend** 目录，先在 MySQL 中执行 **backend/sql/init.sql**（建库建表并插入 3～5 条模拟数据）。用 IDEA 或 Eclipse 打开**根目录**或 backend 目录为 Maven 项目，等待依赖下载完成，运行 **EfficiencyApplication** 的 main 方法。启动成功后监听 **8082** 端口，接口统一前缀为 **/api**。Swagger 文档地址：**[http://localhost:8082/api/swagger-ui/index.html](http://localhost:8082/api/swagger-ui/index.html)**。数据库账号密码为 **root / 123456**，数据库名为 **efficiency_system**。测试账号：**admin / admin123**。

## 3. 后台管理运行方式

进入 **web-admin** 目录，执行 **pnpm install**，再执行 **pnpm run serve**。浏览器打开控制台给出的地址（默认 8083 端口），进入登录页，输入 **admin / admin123** 即可登录并跳转首页。后台仅三页：首页仪表盘（两个 ECharts 图）、待办任务管理、日程安排管理。路由为直接导入，无懒加载。主题为海洋清风色系，图标仅使用约定的 Element 图标。

## 4. 移动端 App 运行方式（HBuilder 浏览器）

移动端为 **Vue2 纯 Web 项目**，用 **HBuilderX** 打开 **mobile-app** 目录，点击「运行 → 运行到浏览器」即可。也可在命令行进入 mobile-app 执行 **pnpm install** 再 **pnpm run serve**（默认约 8081 端口）。底部为**纯文字导航**：待办任务、日程安排、我的。默认登录后进入「待办任务」页，点击「新增」弹出表单，与后台共用同一套后端接口，数据同步。日程的新增会提交「开始时间」「结束时间」为 `yyyy-MM-dd HH:mm:ss` 格式，与后端一致。

## 5. Token 与接口测试

登录接口返回 JWT，前端和 App 将 Token 放在请求头 **Authorization**（可带 `Bearer`  前缀）。在 Swagger 中：先调登录接口拿到 token，再点击右上角 **Authorize**，填入 token（可带或不带 `Bearer` ），之后即可调试需登录的接口。可以把 Token 理解成「临时门禁卡」：登录领卡，后续每次请求都「刷卡」，否则返回 401。

## 6. 常见报错与解决

- **Swagger 打不开**：确认地址为 **[http://localhost:8082/api/swagger-ui/index.html](http://localhost:8082/api/swagger-ui/index.html)**（含 `/api` 前缀）。
- **PathPattern 相关报错**：已在 application.yml 中配置 `spring.mvc.pathmatch.matching-strategy: ant_path_matcher`，若被改过请恢复。
- **数据库连接失败**：确认 MySQL 已启动，账号 **root**，密码 **123456**，并已执行 init.sql 建库。
- **前端白屏**：看浏览器控制台报错，常见为依赖未装完，重新执行 **pnpm install**。
- **App 请求 404 或跨域**：开发时 web-admin 与 mobile-app 通过 devServer 代理转发到后端，请确保后端已启动在 8082 端口。
- **Maven 报错 LocalRepositoryNotAccessible**：若提示无法在 E:repository 创建本地仓库，请在用户目录下配置 Maven 的 settings.xml，将 localRepository 改为你有写权限的路径（如 `C:\\Users\\你的用户名\\.m2\\repository`），或设置环境变量 MAVEN_OPTS 后重试。
