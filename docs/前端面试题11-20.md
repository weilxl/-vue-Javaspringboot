# 前端面试题 11-20

## 11. v-model 的本质是什么？

v-model 是 :value 和 @input 的语法糖。对 input 而言，相当于 :value="xxx" @input="xxx = $event.target.value"，实现双向绑定。在自定义组件里可以指定 model 的 prop 和 event，实现组件级双向绑定。本项目里表单项都用 v-model 绑定 form 对象，改输入即改数据，改数据即更新输入，就像遥控器和电视：一边动，另一边同步。

 

如果面试官问你：“在你的待办事项页面，数据是如何从输入框提交到数据库的？”

“首先，我利用 Vue 的 v-model 实现了表单的双向绑定。当用户在界面上修改内容时，由于数据响应式机制，内存中的 form 对象会实时同步更新。

接着，在用户点击提交时，我直接将这个已经同步好最新数据的 form 对象作为参数，调用封装好的后端接口。后端在接收到这组 JSON 数据后，经过鉴权和业务逻辑处理，最终将其持久化到数据库中。”

## 12. 为什么有时要用 $nextTick？

数据变化后，Vue 的 DOM 更新是异步的，同一轮事件循环里拿到的可能还是旧 DOM。$nextTick 会在下一次 DOM 更新完成后执行回调，此时才能拿到新渲染的节点。本项目在请求完统计数据后，在 $nextTick 里初始化 ECharts，保证图表容器已经按新数据渲染出来。好比等红绿灯：数据更新是「绿灯亮」，$nextTick 是「等车都过了再走」。

## 13. 组件间通信有哪些方式？

父子：props 下发、$emit 上报；跨级：provide/inject 或 Vuex；兄弟：通过共同父组件中转或 Vuex。本项目登录态用 Vuex，父子间用 props 和事件（如 TabBar 的 active 和点击事件）。生活例子：父子像当面交代，兄弟像通过家长传话，全局状态像公司公告栏人人可看。

## 14. 说说 Vue 的生命周期在项目中的使用

created：数据已初始化，可调接口，但 DOM 未挂载；mounted：DOM 已挂载，可做 ECharts.init、滚动等依赖 DOM 的操作；beforeDestroy：可清理定时器、解绑事件。本项目列表页在 mounted 里调 load()，Dashboard 在 mounted 里 loadStats 再在 $nextTick 里画图，符合「先有 DOM 再画图」的顺序。

## 15. 单页应用（SPA）的优缺点？

优点：切换页面无刷新、体验流畅、可做过渡动画；前后端分离、接口复用。缺点：首屏要加载整站 JS、SEO 不友好；需要自己做路由和鉴权。本项目是典型 SPA：一个 index.html，路由切换只换组件不刷新页，适合后台和内部工具，对 SEO 要求不高。

## 16. 如何做请求的「防重复提交」？

常见做法：提交时给按钮加 loading、disabled，或加全局请求队列/锁，同一接口未返回前不再次请求。本项目在登录、新增等操作上用了 :loading="loading"，点击后立刻 loading=true，请求结束再 false，用户不会连续点两次。就像电梯按钮：按一次就亮，再按不会多发一次指令。

## 17. 前端如何配合后端的 401？

在 Axios 响应拦截器里判断 response.status === 401（或 res.code === 401），然后清空本地 token 和 userInfo（如 commit clearAuth），并跳转到登录页（如 router.replace('/login')）。这样任何接口返回未授权，用户都会被统一「请去登录」。本项目已按此实现，保证和后台鉴权策略一致。

## 18. Element UI 表单校验怎么用？

在 el-form 上写 :rules，在 el-form-item 上写 prop 对应 model 的字段，校验规则里可写 required、pattern、validator 等。提交前调用 this.$refs.form.validate(callback)，通过才发请求。本项目登录页对 username、password 做了必填校验，避免空提交。就像填表：必填项不填，提交时会被拦下并提示。

## 19. 为什么要把 API 单独放一个 modules 文件？

集中管理接口地址和请求方式，页面里只调 loginApi、getTodoTaskListApi 等语义化方法，不关心 URL 和 method。这样后端改路径时只改一处，且便于 mock 和单元测试。生活例子：把「所有外卖电话」记在一个本子上，要订餐就翻本子，而不是每间店各记一张纸。

## 20. 如何理解「前端工程化」在本项目中的体现？

本项目体现为：Vue CLI 脚手架、统一的目录（views/api/store/router）、ESLint 规范、pnpm 管理依赖、vue.config.js 配置代理和打包、环境变量区分开发与生产。工程化的目的是可维护、可协作、可复现构建。就像工厂流水线：每个人按工序来，标准统一，质量可控。
