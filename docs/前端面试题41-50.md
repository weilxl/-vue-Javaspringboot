# 前端面试题 41-50

## 41. 什么是「同源策略」？
浏览器限制：只有协议、域名、端口都相同才算同源，否则 AJAX 请求会受跨域限制（默认拿不到响应）。所以前端跑在 localhost:8080、后端在 localhost:8082 时会有跨域。解决：后端 CORS 或前端 devServer proxy。本项目用 proxy，浏览器只请求同源的前端服务器，由前端服务器转发到后端。

## 42. 如何理解「状态提升」？
当多个组件依赖同一份数据时，把数据放到它们共同的父组件，通过 props 下发，通过事件上报修改，保证「单一数据源」。本项目若多个子组件都要用 userInfo，就只从 store 或父组件取一份，不在每个子组件里各存一份，避免不同步。

## 43. 为什么列表渲染建议不用 index 做 key？
增删、排序时 index 会变，导致 Vue 复用错误的 DOM 节点，出现错位或状态错乱。用唯一 id 做 key，diff 时能正确对应「同一条数据」，复用更安全。本项目表格 :key="item.id"，符合最佳实践。

## 44. 如何做「退出登录」？
清空前端登录态：Vuex 里 clearAuth（清 token 和 userInfo），localStorage 里 removeItem；然后跳转到登录页。后端无状态（JWT）时不需要调「注销接口」，只要前端丢掉 token 即可。本项目退出即 commit clearAuth 再 router.replace('/login')，下次请求自然不会带 token，后端会返回 401。

## 45. 什么是「深拷贝」？什么时候需要？
深拷贝是连嵌套对象也复制一份，改拷贝不影响原对象。需要「独立一份数据」时用，如编辑前复制当前行数据到 form，避免直接改原数据。本项目 openEdit 里 this.form = { ...row } 是浅拷贝，若 row 里有嵌套对象且会改嵌套属性，应做深拷贝（如 JSON 解析或 lodash cloneDeep）。

## 46. 如何理解「前端路由」的两种模式？
hash 模式：URL 带 #，如 /#/todo-task，改 hash 不触发整页请求；history 模式：正常路径，如 /todo-task，需服务器配置把所有路径 fallback 到 index.html。本项目后台用 history，移动端用 hash，HBuilder 跑静态资源时 hash 更省配置。

## 47. 什么是「依赖收集」？
Vue 在 getter 里让当前 Watcher 把自己登记到当前属性的 Dep 里，这样谁用到了这个属性就被记下来；将来 setter 触发时，只通知这些 Watcher 更新，没用到的不通知。这就是「依赖收集」，保证精确更新。就像订报纸：只有订了的人才会收到，没订的不发。

## 48. 如何避免「内存泄漏」？
组件里设的定时器、监听器、第三方实例（如 ECharts）在 beforeDestroy 里清除或销毁。若用 addEventListener 就要 removeEventListener，用 setInterval 就要 clearInterval。本项目若 ECharts 实例在组件销毁时未 dispose，可在 beforeDestroy 里 chart.dispose()，避免内存泄漏。

## 49. 如何理解「单向数据流」在 Vuex 中的体现？
Vuex 里 state 只能通过 mutations 改，不能组件里直接改 state；组件通过 dispatch action 或 commit mutation 来改，这样所有变更可追踪、可调试。本项目只用了 mutations（setAuth、clearAuth），没有用 action，但思想一致：改状态走「唯一通道」。

## 50. 前端如何配合后端的「统一返回格式」？
后端通常返回 { code, message, data }，前端在响应拦截器里先看 code：200 正常把 data 给业务，非 200 统一提示 message 并 reject。这样业务里只写 res.data，不用每处都判断 code。本项目拦截器里 if (res.code !== 200) return Promise.reject(...)，请求方直接 .then(res => res.data) 即可。
