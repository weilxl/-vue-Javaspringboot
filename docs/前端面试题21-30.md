# 前端面试题 21-30

## 21. 说说 Vue 中 key 的作用
key 给 v-for 的每个节点一个稳定标识，便于 diff 时复用或重排。用唯一 id 比用 index 更安全：列表增删时 index 会变，可能导致错误复用或状态错乱。本项目表格用 :key="item.id"，保证每条记录对应一个稳定 key。就像班级点名：用学号比用「第几个」更不会搞混人。

## 22. computed 和 watch 的区别与使用场景
computed 是依赖 data 计算出的新值，有缓存，依赖不变就不重算，适合「派生状态」如过滤列表、加总。watch 是监听某个值变化执行回调，适合副作用：请求、弹窗、本地存储。本项目 LayoutView 里用 computed 从 store 取 userInfo 显示在顶栏，没有用 watch，因为只是「读」派生数据。

## 23. 如何理解 Vue 的「单向数据流」？
父通过 props 把数据传给子，子不能直接改 prop，只能通过 $emit 通知父改。这样数据来源清晰，不会出现「不知道谁改的」的问题。本项目 TabBar 接收 active 这个 prop，点击时用 $router.push 跳转而不是改 prop，符合「父管状态、子发事件」的约定。

## 24. 为什么推荐用 const routes 再 new VueRouter({ routes })？
把路由配置抽成常量，便于维护和按模块拆分；同时可以在别处 import routes 做权限过滤、动态路由等。本项目 router/index.js 里先定义 routes 数组再传给 VueRouter，结构清晰，以后若要按角色过滤菜单只需改 routes 或 meta。

## 25. 前端如何「记住」登录状态？
登录成功后把 token 和用户信息存到 localStorage（或 sessionStorage），并写入 Vuex。下次打开页面时，从 localStorage 读 token 初始化 Vuex，路由守卫根据 token 判断是否放行。本项目即如此：setAuth 时写 localStorage，store 初始化时从 localStorage 读，实现刷新不丢登录态。

## 26. 多环境（开发/生产）如何区分接口地址？
用环境变量：Vue CLI 下 .env.development 和 .env.production 里配 VUE_APP_BASE_URL，代码里用 process.env.VUE_APP_BASE_URL。开发时还可直接用 proxy 把 /api 转到本地后端，生产则配成真实域名。本项目开发用 proxy，baseURL 为 /api，由 devServer 转发到 8082。

## 27. 如何避免 ECharts 容器宽高为 0？
确保在 DOM 已渲染且有尺寸时再 init。做法：在 mounted 或 $nextTick 里 init；若容器在 tab 或弹窗里，要在该容器显示后再 init，或监听容器尺寸变化调用 chart.resize()。本项目图表在首页主内容区，mounted + $nextTick 即可。就像量体裁衣：先等人站好再量，不能对着空椅子量。

## 28. 什么是「首屏加载慢」？如何优化？
首屏慢指第一次打开页面要等较多 JS/CSS 下载执行完才可交互。优化：路由懒加载（本项目为求稳用直接导入）、代码分割、压缩、CDN、骨架屏等。本项目体量小，未做懒加载，若页面增多可对非首屏路由恢复懒加载并配合 prefetch。

## 29. 如何理解 Vue 的「渐进式」？
Vue 可以从一个 script 标签开始用，也可以配合路由、状态管理、构建工具做成完整工程，按需选用。本项目用了「Vue + Vue Router + Vuex + Vue CLI + Element」，是较完整的一档，但若只做一个小表单，可以只用 Vue 一个库，这就是「渐进式」的体现。

## 30. 前端安全要注意什么？
不把敏感信息放前端代码里；HTTPS；防 XSS：对用户输入做转义、慎用 v-html；防 CSRF：配合后端 token/cookie 策略；接口鉴权用 token 且注意过期与存储方式。本项目 token 存 localStorage，请求头携带，后端校验，不在前端存密码，符合常见实践。
