# 后端面试题 41-50

## 41. 什么是「ORM」？
Object-Relational Mapping：用对象操作数据库，不用手写 SQL 字符串。MyBatis/MyBatis Plus 把表映射成实体，把方法映射成 SQL，开发者主要写 Java 代码。本项目 TodoTask 实体对应 todo_task 表，TodoTaskMapper 继承 BaseMapper 就有 CRUD，这就是 ORM 的体现。

## 42. 如何理解「事务的传播行为」？
当一个带事务的方法被另一个带事务的方法调用时，事务怎么处理：是加入已有事务（REQUIRED）还是新开一个（REQUIRES_NEW）等。本项目未显式配传播行为，默认 REQUIRED：有事务就加入，没有就新建。若以后有「在事务里调另一个要独立回滚的方法」可考虑 REQUIRES_NEW。

## 43. 什么是「懒加载」？在 MyBatis 里怎么用？
关联数据不立即查，等真正用到再查。MyBatis 里 association/collection 可设 fetchType="lazy"。好处是主表查得快，坏处是可能 N+1。本项目当前无复杂关联，未用懒加载；若以后「用户-待办」一对多且有时只显示用户不显示待办，可考虑懒加载或按需查询。

## 44. 如何做「接口版本」？
常见做法：URL 里带版本如 /v1/todo-task/list，或请求头里带版本，或同一 URL 用参数区分。后端根据版本路由到不同逻辑。本项目未做多版本，若以后要兼容老前端，可加 /v1、/v2 前缀，Controller 或配置里按版本分发。

## 45. 什么是「幂等」？举一个例子。
同一操作执行多次，结果与执行一次相同。例如「根据订单号扣款」：多次用同一订单号请求，只扣一次并返回同一结果，就是幂等。实现可依赖唯一键、状态机或服务端生成的 token。本项目新增待办若重复提交会多条记录，严格场景可给「标题+用户+日期」加唯一约束或前端防重复。

## 46. 如何理解「缓存」？本项目有没有用？
把常用数据放内存（或 Redis），减少查库。本项目未引入 Redis，统计接口每次查库；若访问量大可对 dashboard 结果做短时缓存（如 1 分钟），减少数据库压力。就像把常查的资料放桌上，不用每次都去档案室。

## 47. 什么是「主键」和「索引」？
主键唯一标识一行，一张表一个主键，通常自增 id。索引是辅助查找的结构，可建在非主键列上，加速查询。本项目每张表都有 id 主键自增，未显式建其它索引；上线前可对 username、due_date、start_time 等查得多的列建索引。

## 48. 如何理解「REST 资源」？
URL 表示资源（名词），HTTP 方法表示操作。如 /todo-task 表示待办资源，GET 列表、POST 新增、PUT 更新、DELETE 删除。本项目 /todo-task/list、/todo-task/add、/schedule/delete/{id} 等即按资源+方法设计，清晰易扩展。

## 49. 为什么拦截器里要 excludePathPatterns？
登录、注册、Swagger 等接口不能要求带 token，否则未登录无法登录、文档无法打开。所以在注册拦截器时用 excludePathPatterns 把这些路径排除，只对业务接口做鉴权。本项目排除了 /auth/login、/auth/register 和 Swagger 相关路径，其余一律校验 token。

## 50. 如何理解「无状态」对扩展的意义？
服务端不存会话，请求自带身份（JWT），任何一台机器都能处理任何请求，方便加机器做负载均衡，不用考虑「把同一用户的请求粘到同一台机器」。本项目 JWT 无状态，将来若要上多实例，只需前面挂负载均衡即可，无需 session 共享。
